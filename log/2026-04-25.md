## Metric Validation Notes

- Metrics that worked cleanly:
  - `SR`
  - `BR(class_i)`
  - `RR(B2)`
- Metrics that exposed issues:
  - `RR(B1)` is mechanically zero when the broken subset is defined by B1 failures
  - `Gap_residual` is ambiguous when B3 is deferred
  - `Cost_tok(B3)` cannot be validated without at least one B3 run

## Table Validation Notes

- Benchmark composition table works, but is oversized for single-stratum mini-pilot data
- Breakage/recovery table works and is informative at small scale
- Failure-mode table works, but needs explicit `not_applicable` / `not_run` conventions
